# ADR-003: Select Microservices architecture design

## Status

Confirmed

## Context

[Analysis of the architecture characteristics](https://github.com/canhtuongvinh/cms-system/blob/83cf945c54b5b0655fd3c5fc5a93c0d3dc56463c/SolutionBackground/ArchitecturePatterns.md) required of the system and the possible architecture styles concluded that an Microservices architecture would suit the system, as the least trade-offs than others.

## Decision

Use an Microservices architecture for the the system. The Microservices model was justified through analysis of scalability, integration, and maintainability requirements. It was determined to be the option that meets the systemâ€™s long-term goals while accommodating future growth of CMS company.

**Negative:**
